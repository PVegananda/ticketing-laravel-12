Bab 1

Gambaran Umum Sistem

Apa Itu Manajemen Event?

Fitur inti dari sistem e-ticketing ini

Manajemen Event adalah fitur utama yang memungkinkan Admin mengelola data event. Admin bisa:

  • 1
    Melihat daftar semua event dengan filter, pencarian, dan sorting.
  • 2
    Menambah event baru lengkap dengan tiket (reguler/premium).
  • 3
    Mengedit event — dengan batasan tertentu jika tiket sudah terjual.
  • 4
    Menghapus event — hanya jika belum ada penjualan (Soft Delete).
  • 5
    Melihat detail event di halaman publik.
ℹ️
Fitur ini hanya bisa diakses oleh user yang sudah login dan sudah verifikasi email. Middleware auth dan verified menjaga akses ini.

Migration Database

Struktur Tabel Database

Migration mendefinisikan struktur tabel. Jalankan dengan php artisan migrate

Tabel: events

Schema::create('events', function (Blueprint $table) {
    $table->id();                       // Kolom id auto-increment (Primary Key)
    $table->foreignId('user_id')        // ID admin yang membuat event
          ->constrained()->cascadeOnDelete();
    $table->foreignId('kategori_id')    // ID kategori (Konser, Seminar, dll)
          ->constrained('kategoris')->cascadeOnDelete();
    $table->string('judul');            // Nama / judul event
    $table->text('deskripsi');          // Keterangan panjang event
    $table->string('lokasi');           // Tempat event diadakan
    $table->string('gambar')->nullable(); // Nama file gambar (opsional)
    $table->dateTime('tanggal_waktu'); // Tanggal dan jam event
    $table->softDeletes();              // Kolom deleted_at untuk Soft Delete
    $table->timestamps();              // created_at dan updated_at otomatis
});

Tabel: tikets

Schema::create('tikets', function (Blueprint $table) {
    $table->id();
    $table->foreignId('event_id')->constrained()->cascadeOnDelete();
    $table->enum('tipe', ['reguler', 'premium']); // Hanya boleh 2 nilai
    $table->decimal('harga', 10, 2);    // Harga desimal max 10 digit, 2 di belakang koma
    $table->integer('stok');            // Jumlah tiket tersedia
    $table->softDeletes();
    $table->timestamps();
});
⚠️
Soft Delete ditambahkan belakangan dengan migration terpisah menggunakan $table->softDeletes() di dalam Schema::table() (bukan Schema::create()).

4. Accessor – Atribut Tambahan Otomatis

// getStatusAttribute() -> dipanggil dengan $event->status
public function getStatusAttribute() {
    if ($this->tanggal_waktu->isFuture()) {
        return 'Upcoming';   // Belum berlangsung
    }
    if ($this->tanggal_waktu->copy()->addHours(3)->isFuture()) {
        return 'Ongoing';    // Sedang berlangsung (dalam 3 jam)
    }
    return 'Completed';      // Sudah selesai
}

// getImageUrlAttribute() -> dipanggil dengan $event->image_url
public function getImageUrlAttribute() {
    if (filter_var($this->gambar, FILTER_VALIDATE_URL)) {
        return $this->gambar;  // URL eksternal langsung dipakai
    }
    if ($this->gambar && Storage::disk('public')->exists($this->gambar)) {
        return Storage::url($this->gambar);  // File ada di storage lokal
    }
    return Storage::url('edm_concert.png');  // Fallback: gambar default
}

5. Method hasSales() – Cek Ada Penjualan

public function hasSales() {
    // Cek apakah ada minimal 1 order untuk event ini
    // exists() lebih cepat dari count() karena berhenti di baris pertama
    return $this->orders()->exists();
}

6. Query Scope – Filter Bawaan

// Dipakai dengan Event::upcoming(), Event::ongoing(), Event::completed()
public function scopeUpcoming($query) {
    return $query->where('tanggal_waktu', '>', now());
}
public function scopeOngoing($query) {
    return $query->where('tanggal_waktu', '<=', now())
                 ->where('tanggal_waktu', '>=', now()->subHours(3));
}
public function scopeCompleted($query) {
    return $query->where('tanggal_waktu', '<', now()->subHours(3));
}

EventController – Otak Sistem

Method index() – Daftar Event

URL: GET /admin/events | Daftar event dengan filter, pencarian, sorting, pagination

public function index(Request $request)
{
    $events = Event::with(['kategori', 'tikets'])
        // with() = eager loading: ambil relasi sekaligus, cegah N+1 Query Problem
        ->when($request->kategori_id, function ($query) use ($request) {
            $query->where('kategori_id', $request->kategori_id);
            // Filter: hanya event dengan kategori yang dipilih
        })
        ->when($request->search, function ($query) use ($request) {
            $query->where(function ($q) use ($request) {
                $q->where('judul', 'like', '%' . $request->search . '%')
                  ->orWhere('lokasi', 'like', '%' . $request->search . '%');
            });
            // Pencarian di kolom judul ATAU lokasi (%keyword% = di mana saja)
        })
        ->orderBy('tanggal_waktu', $request->get('sort', 'asc'))
        // Sorting: default asc (terlama), bisa desc (terbaru)
        ->paginate(10); // Pagination: 10 event per halaman

    $kategoris = Kategori::all(); // Untuk dropdown filter
    return view('pages.admin.events.index', compact('events', 'kategoris'));
}

Method store() – Simpan Event Baru

URL: POST /admin/events | Upload gambar, validasi, simpan ke database

public function store(EventFormRequest $request)
{
    // EventFormRequest otomatis validasi SEBELUM masuk ke sini
    // Jika gagal -> redirect balik ke form dengan error
    $validated = $request->validated();

    if ($request->hasFile('gambar')) {
        // store() simpan file dengan nama acak: contoh "events/AbcXyz.jpg"
        $validated['gambar'] = $request->file('gambar')->store('events', 'public');
    } else {
        $validated['gambar'] = 'konser.jpg'; // Gambar default
    }

    $event = Event::create([
        'user_id'       => Auth::id(), // ID admin yang sedang login
        'kategori_id'   => $validated['kategori_id'],
        'judul'         => $validated['judul'],
        'deskripsi'     => $validated['deskripsi'],
        'lokasi'        => $validated['lokasi'],
        'gambar'        => $validated['gambar'],
        'tanggal_waktu' => $validated['tanggal_waktu'],
    ]);

    // Simpan semua tiket yang dikirim sebagai array dari form
    foreach ($validated['tikets'] as $tiket) {
        $event->tikets()->create([
            'tipe'  => $tiket['tipe'],
            'harga' => $tiket['harga'],
            'stok'  => $tiket['stok'],
        ]);
    }

    return redirect()->route('admin.events.index')
                     ->with('success', 'Event berhasil ditambahkan.');
}

Method update() – Simpan Perubahan Event

URL: PUT /admin/events/{event} | Ada proteksi khusus untuk event berpenjualan

public function update(EventFormRequest $request, Event $event)
{
    $validated = $request->validated();

    // PROTEKSI INTEGRITAS DATA:
    // Jika event sudah ada penjualan, tanggal tidak boleh diubah
    // Alasan: tanggal adalah dasar kontrak pembelian tiket
    if ($event->hasSales() &&
        Carbon::parse($validated['tanggal_waktu'])->format('Y-m-d H:i:s')
        != $event->tanggal_waktu->format('Y-m-d H:i:s')) {
        return back()->withErrors([
            'tanggal_waktu' => 'Tanggal tidak dapat diubah karena sudah ada penjualan.'
        ])->withInput();
    }

    // Hanya update gambar jika ada file baru yang diupload
    if ($request->hasFile('gambar')) {
        // Hapus gambar lama dari storage dulu

            Storage::disk('public')->exists($event->gambar)) {

        $validated['gambar'] = $event->gambar; // Pertahankan gambar lama

Method destroy() – Hapus Event (Soft Delete)

URL: DELETE /admin/events/{event} | Event tidak benar-benar terhapus

public function destroy(Event $event)
{
    // Cek ada penjualan? Jika ya, tolak penghapusan
    if ($event->hasSales()) {
        return redirect()->route('admin.events.index')
                         ->with('error', 'Event tidak dapat dihapus karena sudah ada penjualan.');
    }

    // Hapus file gambar dari storage (kecuali gambar default)
    if ($event->gambar !== 'konser.jpg' &&
        Storage::disk('public')->exists($event->gambar)) {
        Storage::disk('public')->delete($event->gambar);
    }

    // Soft delete: mengisi kolom deleted_at dengan waktu sekarang
    // Data TIDAK dihapus dari database!
    $event->delete();

    return redirect()->route('admin.events.index')
                     ->with('success', 'Event berhasil dihapus.');
}

Form Request – Validasi Data

EventFormRequest – Penjaga Gerbang Data

app/Http/Requests/EventFormRequest.php – Semua data divalidasi sebelum masuk ke Controller

public function rules(): array
{
    $rules = [
        'judul'       => 'required|string|max:255',
        // required = wajib, string = teks, max:255 = maks 255 karakter

        'deskripsi'   => 'required|string',
        'lokasi'      => 'required|string|max:255',

        'kategori_id' => 'required|exists:kategoris,id',
        // exists: ID harus ada di tabel kategoris

        'gambar'      => 'nullable|image|mimes:jpg,jpeg,png|max:2048',
        // nullable = opsional, image = file gambar, mimes = format, max = 2MB

        'tikets'         => 'required|array|min:1', // minimal 1 tiket
        'tikets.*.tipe'  => 'required|in:reguler,premium',
        'tikets.*.harga' => 'required|numeric|min:0',
        'tikets.*.stok'  => 'required|integer|min:0',
        'tikets.*.id'    => 'nullable|exists:tikets,id',
    ];

    // Validasi berbeda untuk Tambah vs Edit:
    if ($this->isMethod('post')) {
        $rules['tanggal_waktu'] = 'required|date|after:now';
        // Saat TAMBAH: tanggal harus di masa depan
    } else {
        $rules['tanggal_waktu'] = 'required|date';
        // Saat EDIT: tanggal bebas (event lama boleh sudah lewat)
    }
    return $rules;
}

Aturan Bisnis: Proteksi Event Berpenjualan

Kenapa Ada Batasan Jika Sudah Ada Penjualan?

Konsep data integrity – menjaga agar data tidak saling bertentangan

🚫
Skenario berbahaya tanpa proteksi: Seseorang sudah membeli tiket "Konser Rock, 15 Agustus 2024, 19:00". Lalu admin mengubah tanggalnya ke "20 Desember 2026, 09:00". Pembeli tidak tahu dan sudah membuat rencana berdasarkan tanggal lama. Ini bisa merugikan banyak orang!

Tiga Aturan Proteksi yang Diterapkan

Aturan 1: Tanggal Tidak Bisa Diubah

Input field tanggal diberi atribut readonly di View. Di Controller ada pengecekan ulang menggunakan Carbon::parse() untuk memastikan tanggal tidak berubah meski melewati validasi front-end.

Aturan 2: Tiket Tidak Bisa Ditambah atau Dihapus

Tombol "+ Tambah Tiket" dan tombol "Hapus" di tiap card disembunyikan dengan @unless($hasSales). Di Controller, penghapusan tiket dibungkus dengan if (!$event->hasSales()).

Aturan 3: Event Tidak Bisa Dihapus

Saat tombol "Delete" diklik, Controller mengecek $event->hasSales(). Jika sudah ada penjualan, proses dihentikan dan user diarahkan kembali dengan pesan error.

Cara Sistem Mendeteksi "Ada Penjualan"

// Di Model Event.php:
public function hasSales() {
    // exists() lebih cepat dari count() karena berhenti di baris pertama
    return $this->orders()->exists();
}

// Di Controller:
$hasSales = $event->hasSales(); // true atau false

// Di View (Blade):
@if($hasSales)
    // Tampilkan warning banner
    // Kunci field tanggal (readonly)
    // Sembunyikan tombol tambah/hapus tiket
@endif

Soft Delete – Penghapusan Lunak

Apa Itu Soft Delete?

Teknik "menghapus" data tanpa benar-benar menghapusnya dari database

Hard Delete (Penghapusan Normal)

$event->forceDelete();
-- SQL:
DELETE FROM events WHERE id = 1;
-- Data HILANG SELAMANYA!

Soft Delete (Penghapusan Lunak)

$event->delete();
-- SQL:
UPDATE events SET deleted_at='2026-07-15'
WHERE id = 1;
-- Data MASIH ADA di database!

Cara Kerja Soft Delete di Laravel

  • 1
    Trait SoftDeletes: Tambahkan use SoftDeletes; di dalam Model. Laravel tahu bahwa model ini menggunakan Soft Delete.
  • 2
    Kolom deleted_at: Dibuat dengan $table->softDeletes() di migration.
  • 3
    Query otomatis difilter: Setiap query Eloquent otomatis menambahkan WHERE deleted_at IS NULL. Data yang sudah di-soft-delete tidak muncul!

Model yang Menggunakan Soft Delete di Project Ini

ModelTabelAlasan
EventeventsEvent bisa dipulihkan jika tidak sengaja dihapus
KategorikategorisKategori bisa "dinonaktifkan" tanpa merusak event lama
TikettiketsTiket yang dihapus tetap tercatat untuk audit transaksi
OrderordersData transaksi harus tetap ada sebagai rekam jejak
DetailOrderdetail_ordersDetail pembelian harus tetap ada bersama order-nya

Perintah-Perintah Soft Delete di Laravel

$event->delete();         // Soft delete (isi kolom deleted_at)
$event->forceDelete();    // Hard delete (hapus permanen)
$event->restore();        // Pulihkan data yang sudah di-soft-delete
Event::onlyTrashed()->get();  // Query HANYA data yang sudah dihapus
Event::withTrashed()->get();  // Query SEMUA data termasuk yang dihapus
Event::all();                 // Query normal (otomatis WHERE deleted_at IS NULL)
✓
Manfaat Soft Delete: Data bisa dipulihkan jika admin tidak sengaja menghapus. Riwayat transaksi tetap utuh. Berguna untuk audit log dan laporan historis.

Alur End-to-End

Skenario: Admin Tambah Event Baru

Langkah yang terjadi di balik layar dari klik pertama sampai data tersimpan

  • 1
    Admin klik "+ Tambah Event"
    Browser kirim request GET /admin/events/create.
  • 2
    Middleware verifikasi akses
    Cek: user login? Email terverifikasi? Jika tidak, redirect ke login.
  • 3
    Controller create()
    Ambil semua kategori dari DB, kirim ke View.
  • 4
    View dirender
    create.blade.php dirender menjadi HTML dan dikirim ke browser.
  • 5
    Admin isi form dan klik "Simpan"
    Browser kirim POST /admin/events dengan data form.
  • 6
    EventFormRequest validasi
    Jika gagal: redirect kembali ke form dengan pesan error. Data yang diisi tidak hilang (berkat old()).
  • 7
    Controller store()
    Upload gambar → simpan event → simpan tiket → redirect dengan pesan sukses.
  • 8
    Database menyimpan
    INSERT INTO events ... dan INSERT INTO tikets ....
  • 9
    Admin diarahkan ke daftar event
    Event baru muncul di tabel. Selesai!

Skenario: Admin Hapus Event (Soft Delete)

Alur yang terjadi saat tombol "Delete" diklik

  • 1
    Admin klik "Delete"
    Muncul dialog: "Yakin ingin menghapus event ini?"
  • 2
    Admin klik "OK"
    Browser kirim form POST dengan @method('DELETE').
  • 3
    Controller destroy()
    Cek hasSales(). Jika ada penjualan → redirect error. Jika tidak → lanjut.
  • 4
    File gambar dihapus
    File fisik di storage/app/public/ dihapus.
  • 5
    Soft delete dieksekusi
    $event->delete() mengisi kolom deleted_at. Data tidak hilang dari database.
  • 6
    Event hilang dari daftar
    Semua query otomatis filter WHERE deleted_at IS NULL.
🔍
Buka phpMyAdmin, lihat tabel events — data yang dihapus masih ada di sana, hanya kolom deleted_at-nya terisi timestamp penghapusan.
